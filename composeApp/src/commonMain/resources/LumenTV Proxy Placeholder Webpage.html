视频代理服务接口文档

📋 API 接口列表

GET /proxy

视频内容代理服务，支持范围请求（Range），用于播放器直接访问视频源

请求参数

url string 视频源URL（必填）
headers object 自定义请求头（可选）

请求示例

GET /proxy?url=https://example.com/video.mp4

GET /proxy/m3u8

代理 m3u8 文件，用于 HLS 流媒体播放

请求参数

url string (URL编码) m3u8文件URL（必填）

请求示例

GET /proxy/m3u8?url=https%3A%2F%2Fexample.com%2Findex.m3u8

GET /proxy/cached_m3u8

代理已缓存的 m3u8 文件，通过缓存ID获取

请求参数

id string 缓存ID（必填）

请求示例

GET /proxy/cached_m3u8?id=abc123

GET /video/proxy

Web播放器专用视频代理，支持完整的CORS和视频流处理

请求参数

url string 视频源URL（必填）

请求示例

GET /video/proxy?url=https://example.com/video.mp4

GET 127.0.0.1:9000/player

Web播放器页面，提供完整的视频播放界面

注意，需要lumenTV启动播放服务

请求参数

url string 视频URL（必填）
title string 视频标题（可选）
episode number 集数（可选）

请求示例

GET /player?url=https://example.com/video.mp4&title=视频标题&episode=1

GET /postMsg

向应用发送弹窗消息

请求参数

msg string 消息内容（必填）

请求示例

GET /postMsg?msg=操作成功

OPTIONS /video/proxy

CORS预检请求，用于跨域视频代理

GET 127.0.0.1:9000/health

健康检查端点

注意，需要lumenTV启动播放服务

响应示例

OK

GET 127.0.0.1:9000/info

网页播放器播放的视频信息

注意，需要lumenTV启动播放服务

响应示例

{ "title": "第01集 第 1 集", "url": "https://pans22.s4-tximg.top/pans2/%E7%8E%AB%E7%91%B0%E4%B8%9B%E7%94%9F_01.mp4", "episode": 1 }

💡 使用说明

1. 所有代理接口都支持 Range 请求，确保播放器可以正常拖动进度条

2. 视频代理会自动处理 CORS 和 Referer 头

3. m3u8 代理会自动解码 URL 参数
